#### 🔴 A-01 — `AppScope` 생성자 side-effect 제거

**파일**: `lib/features/app/app_scope.dart`

**문제**  
`InheritedWidget`의 생성자 body에서 `late final` 리포지터리 및 서비스를 직접 생성하고 있다.  
`InheritedWidget`은 불변 값 컨테이너여야 하지만, 현재 구조는 생성자 호출마다 새 인스턴스를 만들고  
`ApiClient`, `PushNotificationService`, `WatchlistController` 등의 리소스를 생성한다.  
`updateShouldNotify`는 항상 `false`를 반환하므로 재생성을 막지만, 이는 보장이 아닌 우연에 의존한다.

**지시**
```
1. AppScope 생성자 body에서 모든 리포지터리/서비스 생성 코드를 제거한다.
2. 모든 의존성을 생성자 파라미터(named required)로 받도록 변경한다.
3. SrLabApp._SrLabAppState.initState() 에서 의존성 객체들을 생성하고 AppScope에 주입한다.
4. _AppScopeDisposerState.dispose()에서 ApiClient, PushNotificationService 등
   Closeable 리소스를 모두 dispose한다.
5. AppScope 생성자는 super() 호출과 파라미터 할당만 수행하도록 순수하게 유지한다.
```

- [ ] 완료

#### 🔴 A-02 — `StockDetailScreen.didChangeDependencies` 중복 로드 방지

**파일**: `lib/features/stock/presentation/stock_detail_screen.dart`

**문제**  
`didChangeDependencies`는 부모 위젯이 rebuild되거나 `InheritedWidget`이 변경될 때마다 재호출된다.  
현재 `_initialized` 가드 없이 매번 `_future = _load()`를 실행하므로 불필요한 API 중복 호출이 발생한다.

**지시**
```
1. _StockDetailScreenState에 bool _initialized = false; 필드를 추가한다.
2. didChangeDependencies 시작 부분에 if (_initialized) return; 가드를 추가한다.
3. _initialized = true; 설정 후 _future = _load(); 를 호출한다.
4. 기존 _reload() 메서드는 _initialized를 변경하지 않고 setState(() { _future = _load(); }) 유지.
```

- [ ] 완료

```
3. 변경 후 기존 push bootstrap 흐름(_didBootstrap 가드)은 그대로 유지한다.
```
