## 현재 확인된 불일치 / 남은 작업

1. **Flutter 폴더 구조 차이**
   - 문서 권장 구조에는 `frontend_app/`가 있지만, 실제 Flutter 앱은 저장소 루트(`lib/`, `android/`, `ios/`, `web/`)에 있습니다.
   - 즉시 동작에는 문제 없지만, 문서와 구조가 다르므로 신규 작업자가 혼동할 수 있습니다.

2. **실제 FCM 미연동**
   - 푸시 구조는 `PushProvider` / `StubPushProvider` 기반으로 준비되어 있습니다.
   - 하지만 Firebase Cloud Messaging 실제 프로젝트 연결, 앱 권한 설정, foreground/background 처리, 실제 토큰 검증은 아직 미구현입니다.

3. **관리자 인증 미구현**
   - 현재 관리자 API는 정식 로그인/세션/JWT가 아니라 `X-Admin-Identifier` 헤더 기반의 최소 운영 방식입니다.
   - 역할(Role) 분리, 계정 관리, 비밀번호 정책, 접근 제어는 아직 없습니다.

4. **관리자 웹 고도화 미완료**
   - `admin_web` 는 운영 검수용 정적 HTML입니다.
   - 디자인 시스템, 실제 인증 연동, 상세 필터/페이징, 운영 대시보드 고도화는 남아 있습니다.

5. **프로덕션 배포 전 추가 작업 필요**
   - 실제 PostgreSQL/운영 환경변수 분리
   - HTTPS / reverse proxy / CORS 정책 정리
   - 에러 모니터링 및 로깅 체계 보강
   - 비동기 배치/큐 기반 푸시 발송 정리
   - 보안 점검 및 운영 계정 보호
   - 앱 릴리즈 빌드/서명/배포 설정

6. **문서와 구현 정렬 작업은 계속 필요**
   - 현재 핵심 상태값(`WAITING`, `TESTING_SUPPORT`, `DIRECT_REBOUND_SUCCESS`, `BREAK_REBOUND_SUCCESS`, `REUSABLE`, `INVALID`) 및 주요 signal type 은 코드/테스트 기준으로 정리되었습니다.
   - 다만 추후 상태 엔진 고도화 시 docs/03과 코드 응답 문구를 계속 함께 맞춰야 합니다.
