# 1. 문서 목적

본 문서는 지지저항Lab 앱 MVP의 화면 구조와 사용자 이동 흐름을 고정하기 위한 문서이다.

이 문서에서 정의하는 범위는 아래와 같다.

- 앱 전체 탭 구조
- 화면 목록
- 각 화면의 목적
- 각 화면의 핵심 구성 요소
- 화면 간 이동 규칙
- 주요 사용자 플로우
- 공통 UI 컴포넌트
- 빈 상태 / 에러 상태 / 로딩 상태

본 문서는 UI 디자인 시안 문서가 아니라,  
**개발자가 실제 화면을 구현할 수 있도록 기능 중심으로 구조를 정의하는 문서**이다.

---

# 2. 앱 구조 개요

지지저항Lab 앱 MVP는 아래 5개 메인 탭으로 구성한다.

1. 홈
2. 관심종목
3. 테마
4. 쇼츠
5. 마이

초기 MVP에서는 이 5개 탭을 기준으로 전체 사용자 경험을 구성한다.

앱의 기본 구조는 아래와 같다.

- **홈**: 오늘 봐야 할 것 요약
- **관심종목**: 내가 등록한 종목 추적
- **테마**: 오늘 강한 테마 및 대장주 탐색
- **쇼츠**: 짧은 영상/요약형 콘텐츠 소비
- **마이**: 설정 및 계정/알림 관리

---

# 3. 전체 탭 구조

## 3-1. 메인 탭 정의

### A. 홈
앱 진입 시 가장 먼저 보게 되는 기본 화면이다.  
오늘의 시장 상태, 오늘의 관찰 종목, 관심종목 신호, 테마 카드 등을 요약해서 보여준다.

### B. 관심종목
사용자가 등록한 종목을 모아보는 화면이다.  
각 종목의 현재 상태, 지지/저항 신호, 알림 설정 여부를 확인할 수 있다.

### C. 테마
오늘 강한 테마와 대장주/후속주를 탐색하는 화면이다.  
이슈와 연결된 종목 탐색용 탭으로 사용한다.

### D. 쇼츠
짧은 영상 또는 요약형 콘텐츠를 모아보는 화면이다.  
유튜브 자동화 프로젝트와의 연동 포인트가 되는 탭이다.

### E. 마이
사용자 계정, 알림 설정, 최근 본 종목, 앱 설정 등을 관리하는 화면이다.

---

## 3-2. 탭 우선순위

탭 우선순위는 아래와 같다.

1. 홈
2. 관심종목
3. 테마
4. 쇼츠
5. 마이

초기 MVP에서는 특히 **홈**과 **관심종목**의 완성도가 가장 중요하다.

---

# 4. 네비게이션 구조

## 4-1. 1차 네비게이션
하단 탭바를 사용한다.

- 홈
- 관심종목
- 테마
- 쇼츠
- 마이

## 4-2. 2차 네비게이션
각 탭 내부에서는 화면 push 방식으로 이동한다.

예:
- 홈 → 종목 상세
- 관심종목 → 종목 상세
- 테마 → 종목 상세
- 쇼츠 → 콘텐츠 상세 또는 외부 링크
- 마이 → 알림 설정 / 계정 화면

## 4-3. 공통 진입 경로
푸시 알림 클릭 시 해당 종목 상세 또는 해당 콘텐츠 화면으로 직접 진입할 수 있어야 한다.

# 6. 홈 화면 명세

## 6-1. 화면 목적
홈 화면은 사용자가 앱을 열었을 때  
**“오늘 무엇을 봐야 하는지”**를 가장 빠르게 전달하는 것이 목적이다.

홈은 정보가 많은 화면이 아니라,  
**즉시 행동을 유도하는 요약 화면**이어야 한다.

---

## 6-2. 핵심 구성 요소

### A. 상단 헤더
- 앱명 또는 로고
- 알림 진입 아이콘
- 검색 진입 아이콘

### B. 오늘의 시장 한 줄
- 시장 분위기를 한 줄로 요약
- 예:
  - 코스피 약보합, 반도체 강세
  - 테마 순환 빠른 장세, 추격 주의
  - 대형주 중심 반등 시도

### C. 오늘의 관찰 종목
- 운영자가 선정한 오늘의 주요 관찰 종목 3~5개
- 카드 형태로 노출
- 각 카드에 포함할 정보
  - 종목명
  - 현재 상태 배지
  - 현재가 또는 등락률
  - 한 줄 해설

### D. 내 관심종목 신호 요약
- 사용자가 등록한 관심종목 중 오늘 신호가 있는 종목 우선 노출
- 예:
  - 지지선 접근 2건
  - 저항선 테스트 1건
  - 이탈 경고 1건

### E. 오늘의 테마 카드
- 강한 테마 3개 정도 노출
- 각 카드 포함 요소
  - 테마명
  - 강도 또는 점수
  - 대장주
  - 한 줄 요약

### F. 최근 분석 업데이트
- 신규 등록된 종목 분석/테마 브리핑 일부 노출
- 홈에서는 2~4개 정도만 노출

### G. 쇼츠/요약 콘텐츠 섹션
- 짧은 영상 또는 1분 요약 콘텐츠 카드
- 탭 전환 없이 일부 맛보기 제공

---

## 6-3. 홈 화면 주요 액션
- 종목 카드 클릭 → 종목 상세 이동
- 관심종목 신호 클릭 → 해당 종목 상세 이동
- 테마 카드 클릭 → 테마 화면 또는 종목 상세 이동
- 검색 아이콘 클릭 → 종목 검색 화면
- 알림 아이콘 클릭 → 알림 목록 또는 알림 설정 화면

---

## 6-4. 홈 화면 우선순위
홈에서 가장 위에 보여야 하는 것은 아래 순서로 한다.

1. 오늘의 시장 한 줄
2. 오늘의 관찰 종목
3. 내 관심종목 신호 요약
4. 오늘의 테마
5. 최근 분석 업데이트
6. 쇼츠 요약

---

# 7. 관심종목 화면 명세

## 7-1. 화면 목적
관심종목 화면은 사용자가 등록한 종목을 모아보고,  
현재 상태와 알림 여부를 한 번에 확인하는 핵심 화면이다.

이 화면은 사실상 앱의 본체 역할을 한다.

---

## 7-2. 핵심 구성 요소

### A. 상단 헤더
- 화면 제목: 관심종목
- 종목 추가 버튼
- 정렬/필터 버튼

### B. 상단 요약 영역
- 총 관심종목 수
- 오늘 신호 발생 수
- 지지선 접근 수
- 저항선 접근 수
- 경고 상태 수

### C. 종목 리스트
각 행 또는 카드에 아래 정보를 포함한다.

- 종목명
- 종목코드(선택)
- 현재 상태 배지
- 현재가 / 등락률
- 주요 레벨과의 거리
- 한 줄 해설
- 알림 ON/OFF 상태

### D. 정렬 기능
- 최근 신호순
- 지지선 접근순
- 저항선 접근순
- 등락률순
- 이름순

### E. 필터 기능
- 전체
- 지지선 접근
- 저항선 접근
- 이탈 경고
- 관찰 보류
- 알림 ON 종목만

---

## 7-3. 관심종목 카드 정보 예시
- 삼성전자
- 상태: 지지선 접근
- 현재가: 66,100
- 지지선까지: 1.4%
- 해설: 박스 하단 재접근 구간

---

## 7-4. 관심종목 화면 주요 액션
- 종목 카드 클릭 → 종목 상세
- 알림 토글 클릭 → 알림 ON/OFF 변경
- 종목 삭제 버튼 클릭 → 관심종목 제거
- 종목 추가 버튼 클릭 → 종목 검색 화면 이동
- 정렬/필터 클릭 → 하단시트 또는 팝업 표시

---

## 7-5. 화면 상태
### 빈 상태
- 아직 관심종목이 없습니다
- 종목을 추가하면 앱이 대신 관찰해드립니다

### 신호 없음 상태
- 오늘은 특별한 신호가 없습니다
- 관심종목 상태는 계속 추적 중입니다

---

# 8. 종목 검색 화면 명세

## 8-1. 화면 목적
사용자가 관심종목으로 추가할 종목을 검색하고 선택하는 화면이다.

---

## 8-2. 핵심 구성 요소
- 검색 입력창
- 최근 검색 종목
- 인기/추천 종목
- 검색 결과 리스트

검색 결과 항목에는 아래 정보를 포함한다.
- 종목명
- 종목코드
- 시장 구분
- 관심종목 추가 버튼

---

## 8-3. 주요 액션
- 검색어 입력
- 결과 목록에서 종목 선택
- 바로 관심종목 추가
- 또는 종목 상세로 이동

---

# 9. 종목 상세 화면 명세

## 9-1. 화면 목적
종목 상세 화면은 한 종목의 현재 가격 상태,  
지지선/저항선, 시나리오, 짧은 해설을 보여주는 핵심 화면이다.

이 화면은 복잡한 HTS 화면이 아니라,  
**판단 보조형 해설 화면**으로 설계한다.

---

## 9-2. 핵심 구성 요소

### A. 상단 헤더
- 종목명
- 종목코드
- 관심종목 추가/제거 버튼
- 공유 버튼(선택)
- 뒤로가기

### B. 기본 가격 정보
- 현재가
- 전일 대비
- 등락률
- 장중 고가/저가(선택)
- 거래량 또는 거래대금(선택)

### C. 상태 배지 영역
- 지지선 접근
- 저항선 테스트
- 지지선 이탈
- 이탈 후 복원
- 반등 성공
- 관찰 보류

### D. 주요 가격 레벨 영역
- 지지선 1
- 지지선 2
- 저항선 1
- 저항선 2

### E. 차트 영역
초기 MVP 차트는 아래 수준만 제공한다.
- 일봉 차트
- 현재가 표시
- 지지선/저항선 수평선 표시
- 최근 추세 확인용 단순 차트

### F. 현재 시나리오 영역
예:
- 기본 시나리오
- 강세 시나리오
- 약세 시나리오

### G. 해설 3줄
- 현재 구간 설명
- 관찰 포인트
- 무효화 조건

### H. 관련 테마/이슈 영역
- 속한 테마
- 대장주 여부
- 관련 테마 카드

### I. 관련 콘텐츠 영역
- 최근 분석 글
- 관련 쇼츠
- 관련 브리핑

---

## 9-3. 종목 상세 주요 액션
- 관심종목 추가/삭제
- 알림 설정 진입
- 관련 테마 클릭
- 관련 콘텐츠 클릭
- 차트 탭 변경(추후 확장)

---

## 9-4. 종목 상세 우선순위
이 화면에서 가장 먼저 보여야 할 것은 아래 순서다.

1. 종목명 + 현재가
2. 상태 배지
3. 지지선/저항선
4. 차트
5. 시나리오
6. 해설 3줄
7. 관련 테마 / 콘텐츠

---

# 10. 테마 화면 명세

## 10-1. 화면 목적
사용자가 오늘 강한 섹터/테마를 빠르게 훑고,  
각 테마의 대표 종목을 확인할 수 있도록 하는 화면이다.

---

## 10-2. 핵심 구성 요소

### A. 상단 헤더
- 화면 제목: 테마
- 정렬 버튼
- 필터 버튼

### B. 테마 리스트
각 테마 카드에 포함할 정보:
- 테마명
- 테마 점수 또는 강도
- 한 줄 요약
- 대장주
- 후속주 1~2개

### C. 정렬 기준
- 강도순
- 최신순
- 거래대금 유입순
- 뉴스 언급순(추후 확장)

### D. 필터 기준
- 오늘 강한 테마
- 최근 3일 강한 테마
- 신규 급부상 테마
- 운영자 추천 테마

---

## 10-3. 테마 카드 예시
- AI 반도체
- 강도: 높음
- 대장주: SK하이닉스
- 후속주: 한미반도체
- 요약: 반도체 장비/메모리 동반 강세

---

## 10-4. 주요 액션
- 테마 카드 클릭 → 테마 상세(추후) 또는 대장주 종목 상세
- 대장주 클릭 → 종목 상세
- 후속주 클릭 → 종목 상세

초기 MVP에서는 별도 테마 상세 화면 없이,  
테마 카드 내 종목 진입 위주로 구성해도 된다.

---

# 11. 쇼츠 화면 명세

## 11-1. 화면 목적
짧은 주식/시장/테마 요약 콘텐츠를 소비하는 화면이다.  
유튜브 자동화와 연결될 수 있는 채널 탭 역할을 한다.

---

## 11-2. 핵심 구성 요소
- 오늘의 시장 요약 영상
- 오늘의 테마 요약 영상
- 종목 빠른 해설 영상
- 카드형 또는 피드형 목록

---

## 11-3. 콘텐츠 유형
- 오늘 시장 1분 요약
- 오늘의 테마 3개

24. 최종 요약

지지저항Lab 앱 MVP의 화면 구조는
홈 → 관심종목 → 종목 상세 흐름이 가장 중요하며,
나머지 테마/쇼츠/마이는 보조 축으로 작동한다.

본 문서의 핵심은 다음과 같다.

홈은 오늘 볼 것을 요약하는 화면이다.

관심종목은 앱의 본체 역할을 한다.

종목 상세는 가격 상태와 해설을 보여주는 핵심 화면이다.

테마와 쇼츠는 탐색과 확장용 탭이다.

마이는 설정과 보조 기능을 담당한다.

즉, 지지저항Lab 앱 MVP는
**“종목 상태를 빠르게 이해하고 반복 확인하게 만드는 구조”**를 기준으로 설계한다.
