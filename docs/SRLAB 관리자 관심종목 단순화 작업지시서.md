# SRLAB 관리자 관심종목 단순화 작업지시서

## 목적
현재 관리자 화면에서 `지지선 상태 강제 수정` 흐름을 제거하고,
운영자가 **종목 검색 → 종목 선택 → 지지선 입력 → 코멘트 입력 → 저장**만으로
홈에 노출되는 관심종목을 관리할 수 있게 UI/로직을 단순화한다.

---

## 첨부파일 확인 결과
아래 구조를 실제 첨부파일에서 확인했다.

- `admin_web/index.html`
  - 탭 구조가 `운영 종목 / 지지·저항 관리 / 지지선 상태 관리 / 홈·테마·콘텐츠` 로 분리되어 있음
  - `지지선 상태 강제 수정` 모달(`stateForceModal`)이 존재함
- `admin_web/assets/app.js`
  - `openStateModal`, `closeStateModal`, `submitStateForceUpdate` 로직이 존재함
  - 현재 저장 흐름이 `종목`, `가격 레벨`, `홈 추천`으로 분리되어 있어 운영 동선이 복잡함
- `admin_web/assets/renderers.js`
  - `renderSupportStatesTable()`에서 `강제 수정` 버튼을 렌더링하고 있음
- `backend_api/app/api/v1/admin.py`
  - `/admin/support-states`, `/admin/support-states/{state_id}/force` 엔드포인트가 존재함
  - `/admin/home-featured`, `/admin/price-levels`, `/admin/stocks`는 이미 존재함
- `backend_api/app/services/stock_view.py`
  - 홈 카드 요약(summary)이 운영자 코멘트 우선이 아니라 support state / 시나리오 기준으로 생성되고 있음

## 이번 작업의 핵심 방향
이번 작업에서는 복잡한 관리 화면을 줄이고,
**홈 관심종목 등록 전용 운영 화면**으로 바꾼다.

운영자가 해야 하는 일은 아래 4개뿐이다.

1. 종목명 일부 또는 종목코드 일부 입력
2. 검색 결과에서 종목 선택
3. 지지선 가격 입력
4. 코멘트 입력 후 저장

즉,
- `지지선 상태 강제 수정` UI 제거
- `종목 신규 생성`을 메인 흐름에서 제외
- `지지/저항 분리 관리`를 메인 흐름에서 제외
- `홈 추천 구성`을 별도 탭에서 또 관리하지 않도록 단일 저장 흐름으로 통합

## 반드시 바꿔야 할 화면 구조

### A. 탭 구조 단순화
기존 탭 중 아래 항목은 제거 또는 비노출 처리한다.

- `탭 C. 지지선 상태 관리`
- `stateForceModal`
- `강제 수정` 버튼 전체

권장 탭 구조:

- `탭 A. 관심종목 관리`
- `탭 B. 테마/콘텐츠 관리`

즉, 지금 사용자 요구와 무관한 `지지선 상태 강제 수정` 흐름은 관리자 화면에서 완전히 안 보이게 한다.

### B. 관심종목 관리 화면의 최종 형태
`탭 A. 관심종목 관리`에는 아래만 남긴다.

#### 상단: 등록/수정 폼
- 검색 입력창 1개
  - placeholder: `종목명 또는 종목코드 입력`
- 자동완성 검색 결과 리스트
- 선택 종목 요약 박스
  - 종목명
  - 종목코드
  - 시장구분
- 지지선 가격 입력
- 코멘트 textarea
- 저장 버튼
- 초기화 버튼

#### 하단: 현재 홈 관심종목 목록
각 항목에 아래 정보 표시
- 종목명
- 종목코드
- 지지선 가격
- 코멘트
- 수정 버튼
- 홈 제외 버튼

정렬은 기존 `home_featured.display_order`를 유지하되,
새로 추가한 종목은 맨 뒤에 붙인다.

## 저장 로직 요구사항
새 화면은 저장 시 아래 동작을 **한 번에** 수행해야 한다.

### 1. 종목 선택 검증
- 사용자가 검색 결과에서 종목을 선택하지 않았으면 저장 불가
- 수동으로 종목코드/종목명을 직접 입력해서 저장하는 방식은 이번 단계에서 메인 UX로 쓰지 않는다

### 2. 지지선 저장 규칙
- 관심종목 화면에서 저장하는 지지선은 `SUPPORT`만 사용한다
- 이 화면에서 사용하는 대표 지지선은 `source_label = 'admin_home'` 로 관리한다
- 동일 종목에 대해 `source_label='admin_home'` 인 활성 SUPPORT 레벨은 1개만 유지한다
- 이미 존재하면 `PUT /admin/price-levels/{id}`
- 없으면 `POST /admin/price-levels`
- `note`에는 코멘트를 저장한다
- `is_active=true`

### 3. 홈 관심종목 등록 규칙
- 저장 시 해당 종목이 `home_featured_stocks`에 없으면 새로 추가한다
- 이미 있으면 기존 항목을 유지한 채 활성 상태만 보장한다
- 저장 시 별도 `홈 추천 저장` 버튼을 또 누를 필요 없게 만든다
- 즉, 관심종목 저장 버튼 한 번으로 홈 노출까지 완료되어야 한다

### 4. 코멘트 저장 규칙
- 코멘트는 아래 2곳에 반영한다
  1. 대표 SUPPORT 레벨의 `note`
  2. `stock.operator_memo`

이유:
- 관리자 재수정 시 폼 재주입이 쉬워야 하고
- 홈 카드 summary에도 운영자 코멘트를 우선적으로 노출하기 위함이다

## 홈 카드(summary) 반영 규칙
`backend_api/app/services/stock_view.py`의 홈 카드 summary 생성 로직을 수정한다.

우선순위는 아래와 같이 바꾼다.

1. `stock.operator_memo` 가 있으면 그것을 summary로 사용
2. 없으면 `source_label='admin_home'` 인 활성 SUPPORT 레벨의 `note` 사용
3. 둘 다 없으면 기존 support state 기반 문구 사용
4. 그래도 없으면 기존 fallback 문구 사용

핵심은,
**홈 카드가 운영자가 입력한 코멘트를 우선적으로 보여주게 하는 것**이다.

---

## 수정 버튼 동작
하단 목록의 `수정` 버튼 클릭 시
해당 항목의 값이 상단 폼으로 다시 채워져야 한다.

채워질 값:
- 선택 종목
- 지지선 가격
- 코멘트

즉, 수정도 팝업/모달 없이 같은 화면 상단 폼에서 처리한다.

---

## 홈 제외 버튼 동작
하단 목록의 `홈 제외` 버튼 클릭 시 아래 처리.

- 해당 종목의 `home_featured_stocks.is_active = false` 처리 또는 목록 갱신 시 제외되게 반영
- 가능하면 이 화면의 대표 SUPPORT(`source_label='admin_home'`)도 함께 `is_active=false` 처리
- 종목 자체(`stocks`)는 삭제하지 않는다
- 감사 로그는 남겨도 되고, 안 남겨도 되지만 기존 패턴을 따를 수 있으면 남긴다

---

## 제거 대상
이번 작업에서 아래 요소는 UI에서 제거한다.

- `지지선 상태 강제 수정` 모달 전체
- `강제 수정` 버튼 전체
- `탭 C. 지지선 상태 관리`
- 메인 흐름에서의 `신규 종목` 버튼
- 메인 흐름에서의 `레벨 유형 선택(SUPPORT/RESISTANCE)`
- 메인 흐름에서의 `근접 기준`, `반등 기준`, `입력 출처` 노출

단, 백엔드 엔드포인트 자체는 이번 단계에서 삭제하지 않아도 된다.
프론트에서만 완전히 비노출 처리해도 된다.

---

## 구현 방식 권장
가능하면 **새 엔드포인트 추가 없이 기존 엔드포인트 조합**으로 끝낸다.

사용 엔드포인트:
- `GET /admin/stocks`
- `PUT /admin/stocks/{stock_id}`
- `GET /admin/price-levels`
- `POST /admin/price-levels`
- `PUT /admin/price-levels/{level_id}`
- `GET /admin/home-featured`
- `PUT /admin/home-featured`

즉,
프론트에서 기존 데이터를 조합하여 `관심종목 저장` helper를 만들고,
한 번의 버튼 클릭으로 내부적으로 여러 API를 순서대로 호출하면 된다.

## 렌더링 규칙

### 목록 카드 표시 내용
각 홈 관심종목 카드에는 최소 아래 정보가 보여야 한다.

- 종목명
- 종목코드
- 지지선 가격
- 코멘트
- 활성 상태 pill
- 수정 버튼
- 홈 제외 버튼

### 검색 UX
- 입력 즉시 필터링
- 종목명 / 종목코드 둘 다 검색 가능
- 검색 결과는 최대 10~12개 정도만 노출
- 검색 결과 선택 전에는 저장 버튼 비활성 또는 저장 시 에러 처리

### 메시지 UX
성공 메시지는 아래처럼 단순하게.
- `관심종목을 저장했습니다.`
- `관심종목에서 제외했습니다.`
- `수정할 관심종목 정보를 불러왔습니다.`

## 테스트 요구사항
최소 아래는 확인한다.

### 수동 확인
1. 관리자 로그인 가능
2. 종목 검색 후 선택 가능
3. 지지선 가격/코멘트 입력 후 저장 가능
4. 저장 즉시 하단 목록에 반영됨
5. 저장 즉시 홈 API 결과(`GET /api/v1/home`)의 featured_stocks에 반영됨
6. 홈 카드 summary가 운영자 코멘트로 노출됨
7. 수정 버튼 클릭 시 상단 폼 재주입됨
8. 홈 제외 버튼 클릭 시 목록에서 제거됨
9. `지지선 상태 강제 수정` UI가 더 이상 보이지 않음
10. 닫히지 않는 모달 문제가 재발하지 않음

### 자동 테스트
가능하면 기존 admin 테스트에 아래 성격의 검증을 추가한다.
- 홈 관심종목 저장 후 `/home` featured_stocks에 포함되는지
- summary가 운영자 코멘트 우선인지

## 완료 기준
아래가 모두 만족되면 완료다.

- 운영자는 종목 검색 → 선택 → 지지선 입력 → 코멘트 입력 → 저장만 하면 된다
- 더 이상 상태 강제 수정 모달이 보이지 않는다
- 홈 관심종목은 별도 탭에서 또 저장할 필요 없이 한 번에 반영된다
- 홈 카드 문구가 자동 상태 문구보다 운영자 코멘트를 우선한다
- 기존 로그인, 테마/콘텐츠 관리, 수동 푸시 기능은 깨지지 않는다
