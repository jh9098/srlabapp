# 1. 프로젝트 한 줄 정의

**지지저항Lab 앱은 한국 개인투자자를 대상으로, 종목의 지지선/저항선 관찰 포인트와 가격 근접 신호를 쉽고 짧게 제공하는 관찰형 주식 앱이다.**

---

# 2. 프로젝트 배경

기존 지지저항Lab은 웹 기반의 콘텐츠 중심 구조를 가지고 있다.  
웹에서는 종목 분석 글, 뉴스 요약, 테마 정보, 종목별 해설 등을 제공할 수 있으나, 사용자가 직접 사이트에 들어와서 확인해야 하는 한계가 있다.

주식 서비스의 특성상 사용자는 다음과 같은 니즈를 반복적으로 가진다.

- 내가 보는 종목이 지금 중요한 가격대에 왔는지 알고 싶다.
- 긴 글보다 지금 어떤 구간인지 빠르게 알고 싶다.
- 관심종목을 앱이 대신 지켜보다가 알려주길 원한다.
- 테마와 대장주를 짧고 쉽게 확인하고 싶다.

따라서 지지저항Lab은 단순 웹사이트 확장이 아니라,  
**“사용자가 매일 반복적으로 여는 관찰형 앱”**으로 재구성할 필요가 있다.

---

# 3. 프로젝트 목적

본 프로젝트의 목적은 아래와 같다.

## 3-1. 웹 중심 구조의 한계 보완
기존 웹은 유입과 콘텐츠 축적에는 유리하지만, 즉시성/반복 사용성/알림 측면에서는 약하다.  
앱은 이를 보완하여 관심종목 관리와 가격 신호 알림을 제공하는 핵심 채널이 된다.

## 3-2. 지지저항Lab의 핵심 역량 제품화
사용자가 수동으로 차트를 해석하지 않아도,  
운영자가 제시한 지지선/저항선을 바탕으로 현재 가격이 어떤 상태인지 직관적으로 전달하는 구조를 만든다.

## 3-3. 반복 방문형 서비스 전환
앱의 핵심 가치는 “읽을거리 제공”이 아니라 “다시 열 이유 제공”이다.  
즉, 알림 → 확인 → 해설 → 재방문의 구조를 만든다.

## 3-4. 향후 확장 가능한 주식 플랫폼의 기반 확보
초기에는 단순 관찰형 앱으로 시작하지만, 이후 아래 영역으로 확장 가능하도록 한다.

- 테마 탐색
- 쇼츠 연동
- 종목 요청
- 프리미엄 신호
- 커뮤니티 기능
- 구독형 수익화

---

# 4. 프로젝트 핵심 방향

지지저항Lab 앱의 방향은 아래 4가지로 정의한다.

## 4-1. 복잡한 차트 앱이 아니라, 쉬운 관찰 앱
이 앱은 전문가용 HTS/트레이딩 앱이 아니다.  
복잡한 보조지표를 다루기보다, 현재 가격이 지지선/저항선과 어떤 관계인지 쉽게 보여주는 데 집중한다.

## 4-2. 콘텐츠 앱이 아니라, 신호 앱
앱의 주 기능은 분석글 목록이 아니다.  
앱의 주 기능은 관심종목의 상태 변화와 그에 대한 짧은 해설이다.

## 4-3. 전종목 실시간 앱이 아니라, 관심종목 중심 앱
초기 MVP는 전체 시장을 실시간으로 다 보여주는 구조가 아니라,  
사용자가 추가한 관심종목과 운영자가 선정한 주요 종목 위주로 운영한다.

## 4-4. 자동화 파이프라인과 연결되는 앱
향후 지지저항Lab 웹, infoall, 유튜브 자동화와 연결하여  
뉴스/테마/분석/쇼츠를 하나의 파이프라인으로 이어질 수 있게 설계한다.

# 5. 핵심 사용자 정의

## 5-1. 1차 핵심 사용자
**국내 개인투자자 중, 차트 해석을 완전히 전문적으로 하지는 않지만, 주요 가격대와 종목 관찰 포인트를 알고 싶어하는 사용자**

특징:
- 차트는 보지만 정교한 기술적 분석은 어려워함
- 관심종목을 자주 확인함
- 짧고 빠른 정보를 선호함
- 앱 알림에 익숙함
- 유튜브/쇼츠/커뮤니티형 정보 소비에 익숙함

## 5-2. 2차 사용자
**주식 유튜브/뉴스/커뮤니티를 자주 보며, 테마/대장주를 빠르게 훑고 싶은 사용자**

특징:
- 시장 이슈에 민감함
- 대장주/테마 위주로 종목을 탐색함
- 긴 리포트보다 빠른 관찰 포인트를 선호함

## 5-3. 3차 사용자
**기존 지지저항Lab 웹 방문자 및 향후 유튜브 유입 사용자**

특징:
- 이미 지지저항Lab 브랜드를 어느 정도 인지할 가능성이 있음
- 앱 설치 후 반복 사용 가능성이 높음

# 6. 해결하려는 문제

본 앱은 아래 문제를 해결한다.

## 문제 1. 사용자가 매번 직접 차트를 확인해야 함
많은 사용자가 관심종목을 직접 검색하고, 가격 구간을 확인하고, 중요한 지점인지 스스로 해석해야 한다.

### 해결 방식
- 관심종목 등록
- 가격 근접 상태 표시
- 지지선/저항선 상태 문구 제공
- 푸시 알림 제공

---

## 문제 2. 정보는 많지만 행동 포인트는 불명확함
뉴스, 커뮤니티, 유튜브 정보는 많지만 “지금 봐야 하는 가격대인지”는 별도로 판단해야 한다.

### 해결 방식
- 종목별 현재 구간 해설
- 지지선/저항선 기반 상태 표시
- 오늘 관찰 종목 카드 제공

---

## 문제 3. 초보 사용자는 차트 해석 자체가 어렵다
캔들, 거래량, 지지/저항의 개념은 알아도 실제로 어떻게 대응해야 할지 어렵다.

### 해결 방식
- 해설 3줄 구조
- 현재 시나리오 요약
- “반응 확인 중 / 반등 성공 / 이탈 후 복원 / 재사용 금지” 같은 상태 문구 제공

---

## 문제 4. 웹은 유입에는 좋지만 반복 사용에는 약하다
웹 분석글은 SEO와 콘텐츠 축적에 유리하나, 앱처럼 매일 열게 만들기 어렵다.

### 해결 방식
- 앱 중심의 관심종목/알림 구조 구축
- 웹은 유입 채널, 앱은 반복 사용 채널로 역할 분리

---

# 7. 제품 포지셔닝

## 7-1. 제품 성격
지지저항Lab 앱은 다음 세 가지 성격을 가진다.

1. **관심종목 관찰 앱**
2. **가격 근접 신호 앱**
3. **짧은 해설 기반 투자 보조 앱**

## 7-2. 하지 않는 것
초기 MVP에서 아래는 지향하지 않는다.

- 실시간 초단타 매매 도구
- 증권사 HTS/MTS 대체 앱
- 자동 매매 연동 앱
- 자유 토론형 대형 커뮤니티
- 과도한 기술지표 나열형 앱

## 7-3. 한 문장 포지션
**“내 종목을 대신 지켜보다가, 지금 중요한 가격 구간에 왔는지 쉽게 알려주는 앱”**

---

# 8. 핵심 가치 제안

지지저항Lab 앱의 핵심 가치 제안은 아래 4가지다.

## 8-1. 지지/저항 관찰 포인트 제공
운영자가 제시한 가격 레벨을 기반으로, 사용자가 복잡한 차트 분석 없이도 주요 가격대를 이해할 수 있다.

## 8-2. 관심종목 추적
내가 등록한 종목을 앱이 대신 관찰하고 상태를 요약해준다.

## 8-3. 가격 근접 알림
지지선/저항선 접근, 이탈, 복원 등 이벤트 발생 시 빠르게 알려준다.

## 8-4. 짧고 쉬운 해설
분석글 전체를 읽지 않아도 지금 어떤 상황인지 1~3줄로 이해할 수 있게 한다.

# 10. MVP 범위 정의

MVP는 반드시 핵심 기능에 집중한다.

## 10-1. 포함 기능

### A. 홈
- 오늘의 시장 한 줄
- 오늘의 관찰 종목
- 관심종목 신호 요약
- 오늘의 테마 카드
- 최근 분석 업데이트 일부

### B. 관심종목
- 종목 검색
- 관심종목 등록/삭제
- 관심종목 리스트 조회
- 종목별 상태 확인
- 알림 ON/OFF

### C. 종목 상세
- 종목명/코드/현재가
- 지지선/저항선
- 차트 기본 표시
- 현재 시나리오
- 해설 3줄
- 관련 테마

### D. 신호/알림
- 지지선 접근
- 지지선 이탈
- 저항선 접근
- 저항선 돌파
- 중복 발송 방지

### E. 테마
- 오늘 강한 테마 목록
- 대장주 / 후속주 표시
- 짧은 테마 해설

### F. 관리자
- 종목 등록/수정
- 지지선/저항선 입력
- 홈 카드 관리
- 테마 관리
- 푸시 발송

---

## 10-2. MVP 핵심 시나리오
사용자는 앱을 설치하고 관심종목을 등록한 뒤,  
지지선/저항선 근접 신호를 받고, 앱에서 현재 상태와 짧은 해설을 확인할 수 있어야 한다.

이 한 줄이 MVP 핵심이다.

# 12. 지지선/저항선 엔진 방향

본 앱의 핵심 차별점은 단순 차트 표시가 아니라,  
운영자가 지정한 가격 레벨의 **상태 변화 해석**에 있다.

초기 MVP에서는 특히 **지지선 상태관리 로직**을 핵심으로 삼는다.

## 핵심 방향
- 지지선은 단순 선이 아니라 상태를 가진 객체로 관리한다.
- 반등 성공 / 이탈 후 복원 / 재활용 가능 / 재활용 금지 같은 상태를 가진다.
- 사용자는 차트 자체보다 상태 문구와 해설 중심으로 이해한다.

이 부분의 구체 구현은 별도 문서인  
`03_지지저항Lab_지지선상태관리_로직명세서.md`에서 정의한다.

# 13. 콘텐츠 전략

앱은 긴 글보다는 짧고 반복적인 소비에 맞춘다.

## 13-1. 콘텐츠 우선순위
1. 상태 문구
2. 해설 1~3줄
3. 종목 카드
4. 테마 카드
5. 쇼츠 연결
6. 긴 분석글 링크

## 13-2. 해설 구조 예시
- 현재 구간 요약
- 관찰 포인트
- 무효화 조건 또는 다음 확인 포인트

예:
- 지지선 근접 구간으로 관찰 가치가 높음
- 이탈 없이 반등 시 1차 유효성 확인 가능
- 종가 기준 하회 시 보수적 접근 필요

# 14. 데이터 운영 방향

초기 MVP는 모든 데이터를 완전 자동으로 처리하는 것보다,  
**운영자가 통제 가능한 구조**를 우선으로 한다.

## 14-1. 초기 원칙
- 종목/지지선/저항선은 관리자에서 직접 입력 가능해야 한다.
- 주가 데이터는 공식 API를 통해 백엔드에서 수집한다.
- 앱에는 가공된 신호/상태 중심으로 제공한다.
- 실시간 데이터는 관심종목 중심으로 관리한다.

## 14-2. 이유
완전 자동화보다 운영 통제가 먼저 확보되어야 서비스 신뢰를 관리할 수 있다.

# 18. 브랜드 방향

지지저항Lab 앱은 아래 인상을 주는 것이 중요하다.

- 어렵지 않다
- 복잡하지 않다
- 빨리 확인할 수 있다
- 믿고 참고할 수 있다
- 차트 해설이 명확하다

즉, 디자인과 문구도 전문가용 매매툴보다는  
**쉽고 빠른 관찰 앱**의 성격이 강해야 한다.

# 19. 개발 원칙

## 19-1. MVP 우선
처음부터 완벽한 주식 플랫폼을 만들지 않는다.  
관심종목 + 신호 + 해설이라는 본질 기능부터 완성한다.

## 19-2. 운영 통제 우선
초기에는 자동화보다 관리자 수동 제어 기능을 우선 확보한다.

## 19-3. 앱 존재 이유 명확화
웹과 똑같은 분석글 목록 앱을 만들지 않는다.  
앱에서만 필요한 가치인 알림, 관심종목, 상태 요약에 집중한다.

## 19-4. 확장 가능한 구조
지지저항Lab 웹, infoall, 유튜브 자동화와 연결될 수 있도록 구조를 닫지 않는다.

# 21. 최종 요약

지지저항Lab 앱 MVP는  
**“관심종목을 등록하고, 지지/저항 가격 구간의 상태 변화를 알림으로 받고, 현재 시나리오를 짧게 이해할 수 있는 관찰형 주식 앱”**으로 정의한다.

초기 개발에서는 아래 3가지를 가장 중요하게 본다.

1. 관심종목 중심 구조
2. 지지선/저항선 상태 기반 신호
3. 짧고 쉬운 해설

즉, 이 앱은 단순 콘텐츠 앱이 아니라  
**“내 종목을 대신 지켜보고 지금 어떤 구간인지 알려주는 앱”**이 되어야 한다.
